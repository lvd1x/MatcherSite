Welcome to Phase One of the RaicesMatcher Project!

About

RaicesMatcher is an algorithm that matches newly admitted students with a host for senior weekend. The project was concieved with the goal of automating the processes of pairing host with students and analyzing data. This is phase one and will take in two CSV files: one of the potential host and another of the incoming students. The files should be dropped into the degsingated area and the RUN bottom should be pressed. The output file will be a CSV of the pairings. Below are the three high level steps of the process.

STEP 1

Collect the data from incoming students and potential host in two CSV files.

STEP 2

Drag and drop the files into the drop section of the site and let the magic happen!

STEP 3

Download the output CSV file of the pairings into your computer.

File Format

In order for the matching algorithm to properly function, the following formatting rules must be followed! Failure to obey the rules will result in an error during the matching process.

  • Commas must be removed from within all cells.
  • Enter must never be used inside a cell.
  • The Host and Student files must follow the format of the examples given.
  • The host file should be named host.csv
  • The student file shoud be named students.csv

Examples of properly formated files are shown below.

Host CSV format:

First Name Last Name Email Location Pronouns Accom 1 Accom 2 Accom 3 Capacity Comments
first 1 last 1 host1@gmail.com Los Angeles Her She Ella TRUE TRUE FALSE 2 Excited to be a host!
first 2 last 2 host2@gmail.com Oakland He His Him FALSE TRUE FALSE 3 Can't Wait!

Student CSV format:

First Name Last Name Email Pronouns Location Comments
first 1 last 1 student1@gmail.com Her she Oakland Excited to see campus!
first 2 last 2 student2@gmail.com He His Him Fresno Can't Wait! also would like accom 1
